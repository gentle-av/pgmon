PostgreSQL Monitor API Документация

Базовый URL: http://localhost:3001/api

Формат данных: JSON

Кодировка: UTF-8

1. Управление серверами (/api/servers)

1.1 Регистрация нового сервера

POST /api/servers/register

Описание: Добавляет новый сервер в систему мониторинга. Автоматически создает учетные данные и конфигурацию опроса. 

{
  "credentialName": "avr18-cred",           // string - уникальное имя учетных данных
  "serverName": "avr18",                     // string - уникальное имя сервера
  "displayName": "AVR18 Database",           // string - отображаемое имя (опционально)
  "host": "localhost",                       // string - хост PostgreSQL
  "port": 5432,                              // integer - порт PostgreSQL (по умолчанию 5432)
  "databaseName": "avr",                     // string - имя базы данных
  "username": "avr",                         // string - имя пользователя БД
  "password": "1",                           // string - пароль (будет зашифрован)
  "sslMode": "disable",                      // string - режим SSL: disable, require, verify-ca, verify-full
  "environment": "production",               // string - окружение: production, staging, development, test
  "description": "Main database",            // string - описание сервера (опционально)
  "connectionTimeoutMs": 5000,               // integer - таймаут подключения в миллисекундах (по умолч. 5000)
  "enabled": true,                           // boolean - включен ли мониторинг
  "priority": 5,                             // integer - приоритет опроса (1-10, чем меньше тем выше)
  "pollingIntervalSeconds": 30,              // integer - интервал основного мониторинга в секундах
  "collectQueries": true,                    // boolean - собирать медленные запросы
  "collectConnections": true,                // boolean - собирать информацию о подключениях
  "collectLocks": true,                      // boolean - собирать информацию о блокировках
  "collectCacheHitRatio": true,              // boolean - собирать кэш-хит-рацио
  "collectUnusedIndexes": false,             // boolean - собирать неиспользуемые индексы
  "collectVacuumStats": true,                // boolean - собирать статистику VACUUM
  "collectAshData": true,                    // boolean - собирать ASH (Active Session History)
  "slowQueryThresholdMs": 1000,              // integer - порог медленного запроса в мс
  "maxSlowQueries": 10,                      // integer - максимальное количество медленных запросов
  "deadTupleRatioThresholdPercent": 10.0     // float - порог мертвых кортежей в процентах
}

1.2 Получить все серверы

GET /api/servers

Описание: Возвращает список всех серверов в системе.

1.3 Получить включенные серверы

GET /api/servers/enabled

Описание: Возвращает только серверы с enabled=true.

1.4 Получить сервер по ID

GET /api/servers/{id}

Параметры пути: id - UUID сервера

1.5 Получить сервер по имени

GET /api/servers/by-name/{name}

Параметры пути: name - server_name сервера

1.6 Обновить сервер

PUT /api/servers/{id}

{
  "displayName": "New Name",                 // string - новое отображаемое имя
  "environment": "staging",                  // string - новое окружение
  "description": "Updated description",      // string - новое описание
  "connectionTimeoutMs": 10000,              // integer - новый таймаут подключения
  "enabled": true,                           // boolean - статус включения
  "status": "online"                         // string - статус: unknown, online, offline, error
}

1.7 Удалить сервер

DELETE /api/servers/{id}

Описание: Удаляет сервер и связанную конфигурацию опроса.

1.8 Включить сервер

POST /api/servers/{id}/enable

1.9 Отключить сервер

POST /api/servers/{id}/disable

1.10 Получить конфигурацию опроса

GET /api/servers/{id}/polling

1.13 Обновить пул соединений

POST /api/servers/{id}/refresh-pool

Описание: Пересоздает пул соединений к серверу (применяет новые настройки).

2. Управление учетными данными (/api/credentials)

2.1 Получить все учетные данные

GET /api/credentials

2.2 Получить учетные данные по ID

GET /api/credentials/{id}

2.3 Создать учетные данные

POST /api/credentials

{
  "name": "avr18-cred",                     // string - уникальное имя учетных данных
  "host": "localhost",                       // string - хост PostgreSQL
  "port": 5432,                              // integer - порт (по умолч. 5432)
  "databaseName": "avr",                     // string - имя базы данных
  "username": "avr",                         // string - имя пользователя
  "encryptedPassword": "1",                  // string - пароль (будет зашифрован)
  "sslMode": "disable",                      // string - disable, require, verify-ca, verify-full
  "enabled": true                            // boolean - активно ли
}

2.4 Обновить учетные данные

PUT /api/credentials/{id}

2.5 Удалить учетные данные

DELETE /api/credentials/{id}

2.6 Проверить подключение

POST /api/credentials/{id}/test

Описание: Проверяет возможность подключения к БД с указанными учетными данными.

2.7 Обновить кэш пароля

POST /api/credentials/{id}/refresh

Описание: Очищает кэшированный пароль, заставляя перечитать его из БД.

3. Метрики и мониторинг (/api/metrics)

3.1 Получить все метрики сервера

GET /api/metrics/all/{serverId}

Описание: Возвращает полный набор метрик: статистика БД, блокировки, индексы, vacuum, подключения.

3.2 Получить активные подключения

GET /api/metrics/connections/{serverId}

Описание: Возвращает список активных подключений с информацией о PID, пользователе, приложении, состоянии, запросе.

3.3 Получить медленные запросы

GET /api/metrics/slow-queries/{serverId}

Описание: Возвращает список медленных запросов на основе pg_stat_statements.

3.4 Получить блокировки

GET /api/metrics/locks/{serverId}

Описание: Возвращает информацию о блокирующих и заблокированных запросах.

3.5 Получить неиспользуемые индексы

GET /api/metrics/unused-indexes/{serverId}

Описание: Возвращает индексы с низким количеством сканирований (idx_scan < 100).

3.6 Получить статистику VACUUM

GET /api/metrics/vacuum/{serverId}

Описание: Возвращает статистику по мертвым кортежам и операциям VACUUM для каждой таблицы.

3.7 Получить активные сессии за период

POST /api/metrics/active-sessions

{
  "serverId": "uuid",                        // string - UUID сервера
  "from": "2026-05-19T22:00:00",             // string - начало периода (ISO формат)
  "to": "2026-05-19T23:00:00"                // string - конец периода (ISO формат)
}

Описание: Возвращает активные сессии из pg_stat_activity за указанный период.

3.8 Получить все активные сессии (кратко)

GET /api/metrics/active-sessions/all

Описание: Возвращает краткую статистику по всем серверам: количество активных и ожидающих запросов.

3.9 Получить сессии по событиям ожидания

POST /api/metrics/active-sessions/by-wait-event

{
  "serverId": "uuid",                        // string - UUID сервера
  "from": "2026-05-19T22:00:00",             // string - начало периода (ISO формат)
  "to": "2026-05-19T23:00:00"                // string - конец периода (ISO формат)
}

Описание: Группирует активные сессии по типу события ожидания (CPU, Lock, IO, LWLock и т.д.).

4. ASH данные (/api/ash)

4.1 Получить ASH данные для графика

GET /api/ash/data/{serverId}?minutes=30

Параметры запроса: minutes - количество минут истории (по умолчанию 30, максимум 1440)

Описание: Возвращает данные для построения графика активных сессий, сгруппированные по минутам и типам ожидания.

5. Алerts (/api/alerts)

5.1 Получить активные алerts

GET /api/alerts

Описание: Возвращает список активных предупреждений: блокировки (HIGH), медленные запросы (MEDIUM), ожидающие запросы >10 (HIGH).

6. Конфигурация (/api/config)

6.1 Получить список БД из конфига

GET /api/config/databases

Описание: Возвращает список баз данных из файла monitoring-config.json5.

6.2 Получить все серверы (через конфиг)

GET /api/config/servers

Описание: Возвращает список серверов из БД.

6.3 Получить порт сервера

GET /api/config/server

Описание: Возвращает порт, на котором запущен мониторинг. 

{"port": 3001}

6.4 Получить настройки мониторинга

GET /api/config/monitoring

Описание: Возвращает глобальные настройки мониторинга (пороги, интервалы по умолчанию).

7. Таблицы (/api/tables)

7.1 Получить самые большие таблицы

GET /api/tables/largest/{serverId}?limit=20

Параметры запроса: limit - количество таблиц (по умолчанию 20, максимум 100)

Описание: Возвращает список самых больших таблиц в БД с размерами таблиц и индексов.

Стандартные коды ответов

  • 200 OK - запрос выполнен успешно
  • 201 Created - ресурс создан
  • 204 No Content - ресурс удален
  • 400 Bad Request - неверный запрос
  • 404 Not Found - ресурс не найден
  • 500 Internal Server Error - внутренняя ошибка сервера
